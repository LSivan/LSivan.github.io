回顾2019

计划完成度

2019的计划包括：

  1. 看书25本（算法方面，redis方面，mysql方面，golang方面，自我/商业管理方面）
  2. 鲨鱼纹，1-6月增肌，6月减脂，7月看效果
  3. 年底存款XXW
  4. 父母的商业养老保险

其中各个项的完成度如下：

1. 看书25本

严重不符合。
别说看完了，光是看了就没有10本，其中《算法》算一本，《明朝那些事儿》算6本，《大秦帝国》算3本，《计算广告》一点点，《高性能MySQL》一点点，然后就没有了。

看《算法》效果并没有想象中的包治百病，面头条的时候，依旧被算法筛选掉了，渣。后面问了haoyuan，他给的答复是多练，没什么技巧，多练才能掌握更多的方法比如什么DFS，DP .etc。

倒是《明朝那些事儿》有点东西，比较有感触的，一个是朱佑樘，好人&好皇帝，影响了我的相处观，“他不仅懂得自己的悲惨，他更能理解敌人的不容易”。且不论放过敌人这件事正确与否，我体会到的东西是，“我知道自己生活过得不算容易，但我的亲人、伴侣过得又能有多简单呢”，所以很多时候有了争执，或者在玩游戏看动漫的时候家里有家务，又或者因为懒、有点累而不想帮家里人做点什么的时候，我都会想到这句话，然后下一个想法就是，”哎，我退让一下吧”/“哎，还是我来吧”。高度概括就是，换位思考这句话的深度落实。

还有一个是于谦，是的，《石灰吟》的作者，大一时一个同学在票圈分享《明朝那些事儿》于谦那一段，我看都没看一眼。直到我今年看完，我也有很强烈的欲望去分享。可以看京师保卫战，他在当时皇帝被俘虏，很多大臣打退堂鼓迁都回南京的时候，毅然决然站出来：“不能回，回了就成为第二个南宋了；我来防御瓦刺”。一个进士出身的文臣，成为了帝都防御战的督战，整饬兵备，部署要害，亲自督战，亲自率兵防御北京城，最后还成功了！！最后的最后，俘虏皇帝成功复辟，他也因为遥尊太上皇和言行过于刚直被嫉恨而在崇文门被处决了。《石灰吟》是他最好的写实，“粉身碎骨浑不怕，要留清白在人间”。于谦给我的影响是，临危受命&责任、担当。所以在今年，在公司这边基本都做到了，有问题基本在1H内给回复或者修复，即便是节假日。

《大秦帝国》就有点，不知道怎么讲，我只能说比起历史书更像是小说吧，所以三卷过去有点看不下去了。不过他那个角度我很喜欢，书名是帝国，但我看了三卷了，只有第一卷的主角有皇帝，第二卷是张仪苏秦，第三卷白起，很明显的传递一个观点，伟大组织之所以伟大脱不开人才，脱不开每一位（强行正确了一波，手动滑稽）。人才也需要合适的舞台，但每个时代每个舞台总有其主角。

2. 鲨鱼纹

今年的健身懈怠了。。。。自从发过挑战彭于晏的票圈后，就开始有点，没那么有动力了，或者说我对肌肉线条的渴求没那么强烈了。更为值得谴责的是，除了健身有点懈怠之外，还染上了熬夜的陋习，严重批评。严重违背了健身的初衷——未来身体的定期投资，强身健体。

3. 年底存款XX

目前存款距离目标还有40K，有点菜。一个客观原因是年终在年后发，计划之外了；还有一个主观上的原因就是，从7月份开始记账停下来了，导致了一个问题就是，花钱没预算，还有就是花了就花了，然后月底还账的时候才发现莫名其妙地怎么账单那么多。但是靠记账就能达到这个目标吗？我觉得不太行。

4. 父母的商业养老保险

问了下相关的人，也在网上做了相关调查。发现这个东西真的有点菜，收益率也就是一个货基，确实没啥必要。还不如靠自己理财呢。

一些思考

学习

“功利性”学习，是最容易持续以恒的，也是我们较常采用的手段。在看一些书籍，文章，看到功利性学习，本来对“功利性”这个贬义词有点排斥，后面对小学到大学，大学再到现在的学习历程进行了思考，发现这居然是对的。。。高考，应试的学习，为了考试而学习，功利吧。上大学，分专业，为了，某个方向的工作而学习，也算是“功利”吧。还有一个问题，学习是为了什么呢？对于我自身，除了满足求知欲之外，我更希望的是，学习是为了解决问题。庄子的话：“生有涯而学无涯”，因而学能尽快用于工作，生活和自我提升的知识，而且有目的之后，就可以量化目标，细化学习流程，并且校验成果，就很稳，这是目前对学习的一些思考。

实例数据共享 & 数据通信

对数据单点的必要性有了一定的认识，假设没有 redis 这个东西，在某些场景下，有这么个缘由：

① 数据没必要每次到DB访问

② 数据缓存到内存

③ 需要多实例抗压 & 备灾

④ 多实例内存的缓存数据不一致

因而需要实例通信，来保证同一服务多实例间数据一致。

还有一个方法就是使用 redis 这种适用于分布式的高性能单点存储，省去了实例间数据同步的繁杂流程，缺点就是要经济成本。确实是充值就能变强。。。

不过这也只是其中一个场景，redis 能应用的场景还很多呢。
